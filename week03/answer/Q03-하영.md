### Q. InnoDB의 클러스터링 인덱스란 무엇이며, 어떤 특징이 있나요?

- PK 순서로 정렬되어 데이터가 저장되는 구조
- 물리적 정렬 순서이므로 테이블 당 하나만 가능
- PK 가 없는 경우
	- NOT NULL 옵션의 유니크 인덱스 중에서 첫 번째 인덱스를 클러스터링 키로 선택
	- 자동으로 유니크한 값을 가지도록 증가되는 컬럼을 내부적으로 추가 후 클러스터링 키로 선택
- PK를 나중에 추가하는 경우에는 새로운 PK를 기준으로 클러스터링 인덱스로 변경하고 데이터 전체를 재정렬

pros
- PK 검색 시 조회 성능이 매우 빠르다.
- 테이블의 모든 세컨더리 인덱스가 PK를 가지고 있기 때문에 커버링 인덱스 검색 시 성능이 좋음

cons
- PK 변경 비용이 매우 크다.
- 랜덤 PK 사용 시 성능 저하

### Q. 세컨더리 인덱스가 데이터를 찾아가는 과정을 설명하세요. 클러스터링 인덱스와의 차이점은 무엇인가요?

세컨더리 인덱스는 PK가 아닌 모든 인덱스를 의미하며, 리프 노드에는 데이터가 아닌 PK를 저장한다.

1. 세컨더리 인덱스(B-Tree) 탐색
2. 탐색된 세컨더리 인덱스의 PK로 클러스터링 인덱스 탐색

만약 조회하고자 하는 컬럼이 모두 세컨더리 인덱스에 포함되어 있다면 클러스터링 인덱스 조회없이 세컨더리 인덱스만 조회한다.
