### Q7. 네임드 락과 메타데이터 락
네임드 락(Named Lock)과 메타데이터 락(Metadata Lock)의 용도를 각각 설명하세요.

네임드 락은 분산 환경에서의 동시성을 제어하기 위해서, <br>
메타데이터 락은 데이터베이스 객체의 구조를 보호하기 위해서 사용된다.

네임드 락은 테이블이나 레코드 같은 DB 객체가 아니라 사용자가 지정한 임의의 문자열에 잠금을 거는 방식이다. <br>
여러 대의 서버가 동시에 실행될 때, 데이터베이스 행 잠금만으로 해결하기 어려운 비즈니스 로직의 동시성을 제어할 때 사용한다. <br>
예를들어, 배치 작업이 중복되는 것을 방지하거나 경합이 심한 로직에서 Redis 등의 세션 공유를 사용할 수 없는 환경에서 대안으로 사용된다.

메타데이터 락은 테이블의 이름이나 구조를 변경하는 경우 획득된다. <br>
이는 사용자가 직접 요청할 수 없으며 MySql 서버가 자동으로 획득한다. <br>
Select, Insert 등이 실행될 때 테이블 구조가 변경되면 안되므로 쿼리 실행중에 Shared MDL 이 걸려 구조 변경을 막는다. <br>
반대로, 스키마 변경 시에는 Exclusive MDL 이 걸려 다른 트랙잭션의 읽기/쓰기를 모두 막는다.
